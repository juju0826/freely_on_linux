# freely_on_linux

Running free software on Linux in a way that is stable, predictable, and sustainable.

在 Linux 上，將自由軟體組裝成一個可以長期、安心運作的系統。

---
## 這個 repository 在做什麼？

這個 repository 記錄的是一條實際走過、並已在真實系統中長期運作的路線。

當我們在 Linux 上自架服務時，重點往往不只是「能不能跑起來」，
而是如何讓自由軟體的組合具備清楚的結構、可預期的行為，以及可持續的維護方式。

這個 repo 關注的不是操作步驟本身，
而是系統在運作過程中，問題為何會出現，以及應該由哪一層負責處理。

## 核心觀念：把事情放在對的層級處理

這個 repo 反覆驗證的一個原則是：各司其職。

系統的穩定性，通常來自於角色分工是否清楚，
而不是單一元件的效能或參數調整。

當每一層只處理自己該負責的事情，
整體行為自然會變得可理解、可預測，也更容易維持。

## 記錄方式

這個 repo 採取漸進式的記錄方式。

每一個被整理出來的經驗，都來自實際運作中的系統，
並經過一段時間的觀察與驗證。

當某個案例的結構與行為已足夠穩定，
它會被獨立整理成文件；
在那之前，README 的角色是說清楚這個 repo 所依循的判斷原則，以及形成這些判斷的理由。
